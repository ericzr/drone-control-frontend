# 智慧无人设施控制后台项目开发文档

## 1. 项目目标

基于 `wemirr-platform` 现有前后端底座，建设一套面向政务协同场景的智慧无人设施控制后台。系统一期以前端先行为主，优先完成控制后台的页面框架、信息架构与演示链路，为后续后端领域建模、算法接入和客户演示提供统一入口。

## 2. 现有资产定位

### 2.1 平台底座

- `wemirr-platform-ui-v4-dev`
  - 作为控制后台前端基础框架
  - 技术栈为 `Vue 3 + TypeScript + Vben + Ant Design Vue + Fast Crud`
- `wemirr-platform-v4-dev`
  - 作为微服务、权限、网关、AI 与插件机制的后端基础
  - 后续承接设备、任务、事件、模型管理等业务服务
- `c124（基于 YOLOv11 的无人机小目标检测系统-深蓝系列）`
  - 当前定位为算法资产包与演示资产
  - 包含训练脚本、权重文件、指标图表、桌面检测应用
  - 对应产品文档 `3.2.7 人工智能模型` 板块中的模型管理、模型评估、演示验证能力来源

### 2.2 关键结论

- 前端一期不切换 UI 技术栈，沿用现有 `Ant Design Vue` 与 `Fast Crud`
- `c124` 一期先以“模型中心展示与演示资产”方式接入，二期再服务化
- 页面建设优先于接口完备，采用“页面骨架 + mock 数据 + 分批联调”推进

## 3. 一期范围

一期聚焦控制后台 Web 端，建设以下模块：

1. 工作台
2. 调度中心
3. 数据中心中的设备管理入口
4. 飞控中心中的任务与计划入口
5. 事件中心
6. 人工智能模型中心

本期暂不展开：

- 客户大屏
- 移动端 APP
- 完整三维地图能力
- 边缘推理与机载部署的真实运行链路

## 5. 开发阶段计划

### 阶段 A：前端框架搭建

目标：完成控制后台新模块菜单、页面骨架与工作台入口。

交付物：

- 新模块路由与菜单
- 控制台总览页面
- 调度、设备、任务、事件、AI 中心页面壳子
- 工作台快捷入口

### 阶段 B：页面细化与原型可用

目标：把页面骨架细化为可演示原型。

交付物：

- 关键指标面板
- 列表页与详情抽屉布局
- 告警复核与任务链路的页面交互
- AI 模型中心中 `c124` 映射展示

### 阶段 C：后端联调

目标：逐步替换 mock 数据，接入真实接口。

优先顺序：

1. 设备管理
2. 任务计划
3. 告警事件
4. 模型管理
5. 模型评估

### 阶段 D：算法平台化

目标：将 `c124` 从桌面演示应用演进为平台内算法模块。

拆解方向：

- 模型文件管理
- 推理服务封装
- 训练指标接入
- 在线演示与结果归档

## 6. 前端开发约定

- 页面统一基于 `Page` 容器开发
- 列表和表单优先使用 `Fast Crud`
- 地图、态势、视频、算法评估等复杂页面使用定制 Vue 页面
- 页面文案先以业务表达准确为优先，后续统一视觉规范
- 接口未就绪时使用页面内静态数据占位，但保留后续替换接口的结构
