# 云界空域OS项目开发文档

## 1. 项目目标

基于 `wemirr-platform` 现有前后端底座，建设一套面向市 / 区县治理场景的低空无人设施协同作业平台。系统一期以前端先行为主，优先完成区域级态势、统一调度、资产纳管、事件闭环与 AI 辅助链路的页面框架与演示链路，为后续后端领域建模、算法接入和客户演示提供统一入口。

## 2. 现有资产定位

### 2.1 平台底座

- `wemirr-platform-ui-v4-dev`
  - 作为控制后台前端基础框架
  - 技术栈为 `Vue 3 + TypeScript + Vben + Ant Design Vue + Fast Crud`
- `wemirr-platform-v4-dev`
  - 作为微服务、权限、网关、AI 与插件机制的后端基础
  - 后续承接设备、任务、事件、模型管理等业务服务
- `c124（基于 YOLOv11 的无人机小目标检测系统-深蓝系列）`
  - 当前定位为算法资产包与演示资产
  - 包含训练脚本、权重文件、指标图表、桌面检测应用
  - 对应产品文档 `3.2.7 人工智能模型` 板块中的模型管理、模型评估、演示验证能力来源

### 2.2 关键结论

- “一网统飞”是整个系统的顶层运行理念，代表区域内无人机、机场、无人车、机器人、摄像头等资产统一纳管、统一调度、统一闭环，并非单独的一个功能模块
- 前端一期不切换 UI 技术栈，沿用现有 `Ant Design Vue` 与 `Fast Crud`
- `c124` 一期先以"模型中心展示与演示资产"方式接入，二期再服务化
- 页面建设优先于接口完备，采用"页面骨架 + mock 数据 + 分批联调"推进

## 3. 一期范围

一期聚焦控制后台 Web 端，建设以下模块：

1. 工作台
2. 指挥中心
3. 调度中心
4. 数据中心中的设备管理入口
5. 飞控中心中的任务与计划入口
6. 事件中心
7. 人工智能模型中心
8. 专题应用样板（首期为林草巡检专题）

- 客户大屏
- 移动端 APP
- 完整三维地图能力

## 5. 开发阶段计划

### 阶段 A：前端框架搭建

目标：完成控制后台新模块菜单、页面骨架与工作台入口。

交付物：

- 新模块路由与菜单
- 控制台总览页面
- 调度、设备、任务、事件、AI 中心页面壳子
- 工作台快捷入口

### 阶段 B：页面细化与原型可用

目标：把页面骨架细化为可演示原型。

交付物：

- 关键指标面板
- 列表页与详情抽屉布局
- 告警复核与任务链路的页面交互
- AI 模型中心中 `c124` 映射展示

### 阶段 C：后端联调

目标：逐步替换 mock 数据，接入真实接口。

优先顺序：

1. 设备管理
2. 任务计划
3. 告警事件
4. 模型管理
5. 模型评估

### 阶段 D：算法平台化

目标：将 `c124` 从桌面演示应用演进为平台内算法模块。

拆解方向：

- 模型文件管理
- 推理服务封装
- 训练指标接入
- 在线演示与结果归档

## 6. 前端开发约定

- 页面统一基于 `Page` 容器开发
- 列表和表单优先使用 `Fast Crud`
- 地图、态势、视频、算法评估等复杂页面使用定制 Vue 页面
- 页面文案先以业务表达准确为优先，后续统一视觉规范
- 接口未就绪时使用页面内静态数据占位，但保留后续替换接口的结构

- 顶栏「指挥中心 / 数据驾驶舱」与场景选择器布局：场景选择器置于左侧并支持下拉展开，避免与视图切换条遮挡；下拉层级高于侧栏卡片。

### 11.2 机巢与监控

- **机巢列表** `/device/dock`：行内「一键起飞」（在线且驻机就绪时显示）→ 参数设置 Modal（目标经纬度、各类高度、失联/航线/命令模式等；图上选点为演示逻辑）→ 确认跳转监控页。
- **机巢监控** `/device/dock-monitor`（`hideInMenu: true`，`activePath: /device/dock`）：四分屏——机巢直播示意、空间地图示意、飞机 FPV 示意、遥测 + 云台虚拟键区。

### 11.3 智能座舱 `/flight/cockpit`

- 右侧云台区增加 **QWEASDC + [Space]** 虚拟键盘，并监听 `keydown` 与点击一致（演示级指令提示）。

### 11.4 业务演示链路（前端 mock）

- 事件列表/详情与 EVT 编号、检测转事件、工单详情页等联动。
- 飞行需求审核、航线申请上下文、调度任务创建与座舱带参跳转等。

### 11.5 AI 模型中心

- 目标检测按输入源（图/视频/摄像头）与光谱波段（可见光/红外/多光谱等）优化信息架构与筛选维度（页面级，待后端契约）。

### 12.2 功能闭环

1. **一键起飞 → 任务**：确认后是否创建飞行任务、列表回显、返航/中断状态回写机巢行。
2. **视频与直播**：机巢/飞机画面 URL、鉴权、断线重连、多路切换组件抽象。
3. **键盘操控安全**：座舱/监控页 `keydown` 与输入框、浏览器快捷键冲突；急停优先级、仅在有控制权时启用。

### 12.3 工程与体验

1. **文档**：保持本文与路由表一致；可选新增 `ROUTES.md` 附录列出业务路由、`hideInMenu` 与 mock 状态。
2. **代码质量**：合并重复 `onMounted`、定时器统一在 `onBeforeUnmount` 清理；地图 destroy/init 路径回归测试。
3. **i18n / a11y**：控制台类页面中文硬编码，按需抽 key 与焦点管理。
4. **测试**：核心链路 smoke（机巢→监控、指挥中心终端切换、事件→工单）。

### 12.4 建议 Sprint 顺序（全盘优化）

1. **文档与路由索引**：更新本文 +（可选）`ROUTES.md`。
2. **机巢飞控**：API 契约 → 起飞参数校验 → 监控页状态机；可选地图选点。
3. **指挥中心**：视频占位组件抽象；地图与终端视图状态机；窄屏回归。
4. **座舱与操控**：虚拟键与云台/变焦联动；键盘冲突与急停。
5. **去 mock**：按 §10 原顺序设备 → 任务 → 事件 → 模型。
