# 无人机控制台前端优化迭代计划

本文档与根目录 [DEV_PLAN.md](../../DEV_PLAN.md) **§12、§14** 配套：**DEV_PLAN 偏业务与接口闭环**，本文偏**前端工程结构、性能、一致性与可维护性**。实施时与 §14.6 波次穿插即可。

---

## 1. 目标与原则

- **不阻塞业务 Wave**：先做「泄漏/安全/错误体验」类 P0，再做「抽象复用/性能」，最后做「i18n/全量 a11y」。
- **与路由模块对齐**：`drone-control.ts`、`situation-board.ts` 等新增子路由时，同步 [ROUTES.md](../ROUTES.md) 与 smoke 步骤。
- **单路径可验证**：每项优化尽量对应可手测或自动化的一条路径（见 [SMOKE.md](./SMOKE.md)）。

---

## 2. 架构与代码组织

| 优先级 | 项 | 说明 |
|--------|-----|------|
| P0 | 视图与 API 分层 | `views/wemirr/drone-control/**` 内复杂页：请求与协议解析收口到 `api/drone/*` 或 `_services/*`，页面只编排 UI 与状态 |
| P0 | 组合式函数边界 | 地图、视频、安全键盘、定时遥测等已有或新增 composable：**单一职责**、导出清晰类型；避免页面内 500+ 行混杂协议与样式 |
| P1 | 领域类型收敛 | 在 `views/.../drone-control/types/` 扩展枚举与 DTO，与后端 OpenAPI/Swagger 对齐后替换 `any` 与魔法字符串 |
| P1 | 常量与配置 | 流地址、地图 key、轮询间隔等：**环境变量 + 运行时配置**（禁止散落硬编码） |
| P2 | 功能切片目录 | 同一业务域（如机巢）的 Modal、子组件、store 可收拢到 `device/dock/` 子目录，避免根目录堆积 |

---

## 3. 地图、视频与大屏

| 优先级 | 项 | 说明 |
|--------|-----|------|
| P0 | 地图生命周期 | 所有 `useMap` 使用方：`onBeforeUnmount` / 路由离开时 **显式 destroy**；场景切换防重复 init（与 DEV_PLAN §12.3 一致） |
| P0 | 视频组件契约 | 定义统一 `VideoPanel` props（src、poster、auth、onError、retry）；多路布局只组合、不复制播放器逻辑 |
| P1 | 大屏性能 | `situation-board` / `DataScreenView`：大数据列表虚拟滚动、图表按需挂载、轮询合并与页面可见性（`document.visibilityState`） |
| P1 | 图层与瓦片 | 底图切换、终端切换时取消未完成的 tile 请求（若地图库支持） |
| P2 | 二三维扩展 | 若引入三维：封装适配层，避免业务页直接依赖具体引擎 API |

---

## 4. 交互、安全与可访问性

| 优先级 | 项 | 说明 |
|--------|-----|------|
| P0 | 全局热键策略 | 复用并扩展 `composables/useSafeFlightKeys`：Modal/Drawer/输入焦点、Ctrl/Cmd 组合键、**紧急操作优先** |
| P0 | 防重复提交 | 起飞、返航、紧急类操作：`loading` + 禁用按钮 + 请求 idempotency（与后端约定时带上 clientRequestId） |
| P1 | 焦点管理 | 大屏内 Tab 顺序、打开弹窗后焦点 trap（Ant Design Vue Modal 默认行为上再校验座舱页） |
| P1 | 对比度与字号 | 指挥/驾驶舱主题：关键告警满足 WCAG 对比度抽查；最小字号约定 |
| P2 | 全量 i18n | 按模块抽 key；优先导出类、错误提示、空态文案 |

---

## 5. Fast Crud 与列表体验

| 优先级 | 项 | 说明 |
|--------|-----|------|
| P0 | 详情策略统一 | 遵循 [CRUD-DETAIL-PATTERN.md](./CRUD-DETAIL-PATTERN.md)；全页与抽屉跳转参数一致 |
| P1 | rowHandle 与路由 | 各设备/事件 `crud.tsx`：详情、监控、座舱跳转路径统一常量 |
| P1 | 加载与错误 | 列表请求失败：message + 保留筛选条件；避免静默失败 |
| P2 | 列宽与缓存 | 用户列设置 localStorage 策略（与平台其它模块一致即可） |

---

## 6. 状态、缓存与数据获取

| 优先级 | 项 | 说明 |
|--------|-----|------|
| P0 | 会话级演示数据 | `sessionStorage` 类方案（如起飞任务）：文档写明生命周期；接真 API 后改为服务端状态或短期 token |
| P1 | 列表与详情一致性 | 从详情返回列表时：必要时 invalidate 或乐观更新，避免陈旧行状态 |
| P1 | WebSocket / SSE | 若后端推送遥测、任务状态：统一连接管理与重连退避，避免每页各建一套 |
| P2 | 请求去抖与节流 | 地图选点、搜索框、座舱指令等：与产品确认延迟参数 |

---

## 7. 质量保障

| 优先级 | 项 | 说明 |
|--------|-----|------|
| P0 | Smoke 文档 | [SMOKE.md](./SMOKE.md) 随新链路增补；发版前按路径手测 |
| P1 | E2E 选型与最小集 | 选定 Playwright 或其它；先覆盖：机巢→监控、指挥终端切换、事件→工单 |
| P1 | 类型检查 | 在 CI 或本地脚本中对 `views/wemirr/drone-control` 增量 `vue-tsc`，逐步扩大至全包 |
| P2 | 视觉回归 | 大屏页可选截图对比（低频、大版本） |

## 8. 构建与依赖健康

| 优先级 | 项 | 说明 |
|--------|-----|------|
| P1 | 分包与懒加载 | 地图、视频、大图表库：**路由级动态 import**，控制首包体积 |
